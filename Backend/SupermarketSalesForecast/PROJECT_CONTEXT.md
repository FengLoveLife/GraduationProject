# 🛒 中小型超市日销量预测与智能进货系统 - 全局架构与开发上下文指南

## 1. 项目概述与核心定位

本项目是一个为中小型超市量身定制的现代化 ERP 与商业智能（BI）系统。
与传统仅负责记录数据的 ERP 不同，本系统的核心价值在于**“数据驱动决策”**：通过精准记录商品档案与库存台账，收集销售流水，最终利用 Python 机器学习算法结合环境因子（天气、节假日），对未来的日销量进行精准预测，并自动生成智能进货建议，解决超市“压货”与“缺货”的痛点。

## 2. 全局技术栈与【不可违背的开发铁律】
### 2.1 前端技术栈

* **核心框架**：Vue 3 (Composition API / `<script setup>`) + Element Plus + Axios + Echarts。
* **🚨 绝对禁令（前端）**：**项目中绝对不允许出现任何 TypeScript 代码！** 所有的代码必须使用原生 JavaScript (纯 JS) 编写。
* **UI 交互规范**：采用极简现代的 B 端风格，模块多用去边框带淡阴影的卡片 (`el-card`) 布局。重要状态需有颜色视觉引导（如：库存告急用红色，进货用绿色，警告用奶酪黄）。

### 2.2 后端技术栈 (包名：`com.saul`)

* **核心框架**：JDK 17/21 + Spring Boot 3.x + MyBatis-Plus + MySQL 8 + EasyExcel + JWT。
* **🚨 绝对禁令（后端）**：**禁止在 XML 中编写复杂的 SQL 连表查询（JOIN）！** 所有跨表关联数据（如流水表关联商品名称），必须通过 `Java 8 Stream API` 在内存中进行高效组装，以降低数据库耦合。
* **架构分层**：严格遵守 `Controller -> Service -> Mapper`。数据传输严格区分 `DTO` (入参) 和 `VO` (出参)。
* **接口规范**：所有接口必须使用统一的 `Result.success()` 或 `Result.error()` 进行包装。

### 2.3 算法与预测技术栈（分阶段演进）

* **前期（当前阶段）**：使用 Java 构建完善的数据底座（特征工程），暂用规则或 Mock 数据模拟预测结果，优先跑通前后端业务闭环。
* **后期（计划阶段）**：使用 Python (Pandas 数据清洗 + Scikit-Learn/LightGBM 树模型) 训练预测模型，并通过 FastAPI 包装成微服务供 Java 端调用。

## 3. 核心业务模块与数据库设计哲学

本系统的数据流转逻辑极其严密，遵循**“主数据只读，库存变动全靠流水驱动”**的财务级安全原则。

### 3.1 商品与库存底座（已完成）

* **商品分类 (`product_category`)**：支持无限极分类，采用内存组装树形结构。
* **商品档案 (`product`)**：记录商品基础信息（`purchase_price`, `sale_price`, `safety_stock`）。**注意：编辑商品接口已在 DTO 层屏蔽了 `stock` 字段，严禁直接修改库存！**
* **库存流水 (`inventory_log`)**：任何库存变化必须向此表插入记录。支持 4 种类型：1-进货，2-销售，3-损耗盘亏，4-手工调整。防呆设计严密（如盘亏只能为负数），操作人从 JWT Token 自动解析。

### 3.2 销售数据中心（当前开发核心）

本模块是 AI 预测的数据源（提供目标变量 $Y$），必须采用**企业级主从表快照设计**：

* **`sales_order` (主表)**：记录订单号、总金额、时间。
* **`sales_order_item` (明细从表 - 极度重要)**：不仅记录关联的 `product_id`，还**必须冗余快照**当时的 `purchase_price`（进货价）、`unit_price`（售价）、`product_name` 等，并包含 `is_promotion`（是否促销）字段，为 AI 提供精准历史切片，同时支撑系统计算历史毛利。

### 3.3 预测环境因子库（AI 特征工程）

本模块负责提供影响销量的外部特征变量（提供特征向量 $X$）：

* **`calendar_factor` (日历因子表)**：以 `date` 为主键，记录每天的星期几、是否周末、是否法定节假日、天气情况（晴/雨/雪）等。通过解耦交易表和日历表，大幅减少数据冗余。

## 4. 当前项目进度与待办任务

### ✅ 已完美完成的模块：

1. 全局配置：JWT 鉴权拦截器、统一异常处理、跨域配置、本地图片虚拟路径映射上传。
2. 商品档案管理模块（含商品与分类的完整 CRUD，基于 EasyExcel 的条件导出）。
3. 库存预警与监控大屏（基于 Stream 流算力的 KPI 统计、图表数据结构）。
4. 库存变动流水台账（含极具企业级交互的手工盘点弹窗、防篡改机制及流水导出）。

### 🚀 正在进行/即将开始的任务（Next Steps）：

1. **销售数据中心基础建设**：基于已确认的表结构（`sales_order`, `sales_order_item`, `calendar_factor`）生成后端的 Entity、Mapper、Service 骨架。
2. **销售日结单导入业务**：实现读取每日销售 Excel，根据数据批量插入主从表、扣减 `product` 表库存，并自动生成 `Type=2` 的 `inventory_log` 记录。
3. **因子表数据维护**：实现 `calendar_factor` 的数据初始化（自动推算周末/节假日）。
