# 中小型超市日销量预测与智能进货系统


---

## 项目简介

本系统面向中小型超市，通过 **LightGBM 机器学习模型** 对商品日销量进行预测，并基于预测结果结合库存安全水位，实现 **智能进货建议** 的自动生成与进货单全流程管理。系统将数据分析与日常进货决策深度结合，帮助中小超市老板降低缺货风险、减少盲目囤货，提升资金周转效率。

### 核心价值

- **销量预测**：基于历史销售数据、日历因子（节假日、天气、星期）训练 LightGBM 模型，提供未来 7 天的商品日销量预测
- **智能进货**：红黄绿三灯策略自动识别缺货风险，结合补货周期与安全库存生成精准进货建议
- **进货全流程**：支持进货建议→调整→生成进货单→已下单→确认入库的完整业务流转
- **库存监控**：实时监控库存健康度，支持逾期预警与紧急补货导出

## 功能模块

### 1. 首页大盘

- 今日销售额、毛利润、进货成本、客单量等核心 KPI
- 近 30 天销售趋势折线图
- 商品品类销售占比饼图
- 快速入口导航

### 2. 基础数据管理

- **商品分类管理**：支持配置各分类的补货周期（天），该周期直接影响进货建议量的计算
- **商品管理**：商品信息、进货单价、安全库存的维护与查看

### 3. 销售管理

- **销售日结导入**：支持 Excel 文件导入日销售流水，系统自动扣减库存并记录库存流水
- **销售记录查询**：支持按日期、商品、分类多维度筛选
- **销售分析**：销售趋势图、品类占比、毛利分析
- **销售日历**：以日历视图展示每日销售额

### 4. 销量预测

- **预测看板**：预测库存状态分布饼图（红/黄/绿三灯）、7 天预测销量走势、分类汇总
- **销量预测明细**：展示所有商品的预测日均销量、当前库存、安全库存、库存状态  
  - 红灯（库存告急）：当前库存 ≤ 安全库存  
  - 黄灯（库存紧张）：当前库存 ≤ 安全库存 + 周期销量 × 30%  
  - 绿灯（库存充足）：其余情况  
  - 支持按库存状态筛选与预测日期切换

### 5. 智能进货（核心模块）

**进货建议**
- 一键生成进货建议：系统自动扫描红灯/黄灯商品，结合三档日均权重和补货周期计算建议进货量
- 可对每件商品的进货量进行调整，修改即时持久化到数据库
- 分页展示（默认 10 条/页），支持红/黄灯筛选
- 确认生成进货单弹窗展示完整进货明细，供用户二次核对
- 支持导出 CSV 进货清单（含商品名、进货量、单价、小计）
- 移除不需要进货的商品（标记为已忽略）

**进货记录**
- 完整进货单状态流转：**待确认 → 已下单 → 已完成** / **已取消**
- 确认进库后自动更新商品库存并记录库存流水
- 预计到货日逾期提醒：已下单状态且预计到货日已过，自动标红显示"逾期"
- 支持按状态、日期范围筛选，进货单详情弹窗展示商品明细

### 6. 库存管理

- **库存监控中心**：实时展示在售商品数、库存告急 SKU、库存预警 SKU、平均库存健康度
- 支持导出紧急补货单（Excel）
- **库存流水**：完整记录每次进货入库与销售出库的库存变动明细

### 7. 系统设置

- 个人信息管理
- 操作日志查询

## 快速启动

### 环境要求

- JDK 17+
- Node.js 18+
- Python 3.10+
- MySQL 8.0

### 1. 数据库初始化

```sql
-- 创建数据库
CREATE DATABASE supermarket_forecasting_db CHARACTER SET utf8mb4;

-- 执行建表脚本
source Backend/SupermarketSalesForecast/src/main/resources/db/table.sql

-- 执行初始数据
source Backend/SupermarketSalesForecast/src/main/resources/db/mock.sql

-- 执行安全库存初始化
source db/safety_stock.sql
```

### 2. 启动 Python 预测服务

```bash
cd PythonService
pip install -r requirements.txt

# 启动服务（首次启动若无模型文件，自动训练后启动）
python -m uvicorn forecast.api_service:app --host 0.0.0.0 --port 8000
```

服务启动后访问 `http://localhost:8000/docs` 查看 API 文档。

### 3. 启动 Java 后端

```bash
cd Backend/SupermarketSalesForecast
# 修改 application.yml 中的数据库连接配置
mvn spring-boot:run
```

后端默认端口：`8080`

### 4. 启动前端

```bash
cd Frontend/supermarket
npm install
npm run dev
```

前端默认访问地址：`http://localhost:5173`

## 核心业务流程

### 销量预测流程

```
销售数据导入 → sales_order_item 表
                    ↓
Python 预测服务读取历史销量 + 日历因子
                    ↓
LightGBM 滚动预测（逐日推进7天）
                    ↓
预测结果写入 forecast_result 表
                    ↓
Java 后端读取预测结果 + 商品库存
                    ↓
前端展示预测数据（含三灯库存状态）
```

### 智能进货流程

```
点击"生成进货建议"
        ↓
批量预加载：历史日均(30天) + 预测日均(7天)
        ↓
三档加权计算日均销量
（≤7天:100%预测 | 7-14天:60/40 | ≥15天:30/70）
        ↓
红灯扫描：当前库存 ≤ 安全库存 → 必须进货
黄灯扫描：当前库存 ≤ 安全库存 + 周期销量×30% → 顺带进货
        ↓
计算建议进货量 = 目标库存 - 当前库存
目标库存 = 日均销量 × 补货周期 + 安全库存
        ↓
用户查看/调整进货量（实时持久化）
        ↓
确认生成进货单（弹窗展示完整明细）
        ↓
标记"已下单"（货在路上）
        ↓
确认入库 → 库存自动增加 + 记录流水
```

## 安全库存计算说明

安全库存基于 **95% 服务水平**（Z=1.65）的统计公式计算：

```
安全库存 = CEIL(1.65 × 日销量标准差 × √补货周期天数)
```

同时施加约束：
- 下限：不低于 `0.8 × 日均销量`（保底）
- 上限：不超过 `3 × 日均销量`（防畸高）

安全库存基于 2025-09-01 至 2026-04-11 的实际销售数据计算，已通过 `db/safety_stock.sql` 写入数据库。

## 开发说明

- Python 预测服务与 Java 后端通过 HTTP 接口通信，**解耦**设计，可独立升级模型
- 预测结果写入 `forecast_result` 表，Java 后端直接读库，无需每次请求都调 Python
- 进货建议生成时采用 **批量预加载** 消除 N+1 查询，整个生成流程固定只有 4 条 SQL
