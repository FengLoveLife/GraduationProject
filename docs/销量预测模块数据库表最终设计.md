## 一、现有表结构分析

### 1.1 `forecast_result` 表（已存在）

```sql
CREATE TABLE `forecast_result` (
    `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '主键ID',

    -- 预测目标
    `forecast_date` DATE NOT NULL COMMENT '预测的目标日期',
    `product_id` BIGINT NOT NULL COMMENT '商品ID',

    -- 冗余字段（减少JOIN查询）
    `product_code` VARCHAR(50) NOT NULL COMMENT '商品编码(冗余)',
    `product_name` VARCHAR(100) NOT NULL COMMENT '商品名称(冗余)',
    `category_id` BIGINT DEFAULT NULL COMMENT '分类ID(冗余)',
    `category_name` VARCHAR(50) DEFAULT NULL COMMENT '分类名称(冗余)',

    -- 预测输入特征
    `weather` VARCHAR(20) DEFAULT NULL COMMENT '预测时使用的天气条件',
    `is_holiday` TINYINT DEFAULT 0 COMMENT '预测时是否节假日: 0否 1是',
    `is_weekend` TINYINT DEFAULT 0 COMMENT '预测时是否周末: 0否 1是',

    -- 预测结果
    `predicted_quantity` INT NOT NULL COMMENT '预测销量',
    `confidence` INT DEFAULT NULL COMMENT '预测置信度(0-100)',

    -- 实际对比
    `actual_quantity` INT DEFAULT NULL COMMENT '实际销量',
    `error_rate` DECIMAL(5,2) DEFAULT NULL COMMENT '预测误差率(%)',

    -- 审计字段
    `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    `update_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,

    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_forecast_product_date` (`forecast_date`, `product_id`),
    KEY `idx_forecast_date` (`forecast_date`),
    KEY `idx_product_id` (`product_id`),
    KEY `idx_category_id` (`category_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='销量预测结果表';
```

## 二、字段补充建议

### 2.1 建议新增字段

经过分析，建议补充以下字段：

```sql
-- 在 forecast_result 表中新增以下字段

-- 1. 预测时的当前库存快照（用于计算进货建议）
`current_stock` INT DEFAULT NULL COMMENT '预测时的当前库存(快照)',

-- 2. 预测时的安全库存快照
`safety_stock` INT DEFAULT NULL COMMENT '预测时的安全库存(快照)',

-- 3. 预测批次号（标识同一次批量预测）
`batch_no` VARCHAR(32) DEFAULT NULL COMMENT '预测批次号',
```

### 2.2 补充原因说明

| 字段 | 补充原因 |
|------|----------|
| `current_stock` | 记录预测时的库存状态，生成进货建议时需要此数据。如果不记录，后续库存变化后无法追溯当时的库存情况 |
| `safety_stock` | 同上，安全库存可能被用户修改，需要快照记录 |
| `batch_no` | 批量预测时，标识哪些预测记录是同一次任务产生的，便于管理和追溯 |

### 2.3 补充后的完整表结构


## 三、是否需要其他表？

### 3.1 预测模块表结构评估

| 可能需要的表 | 是否必要 | 建议 |
|--------------|----------|------|
| `model_config` 模型配置表 | ⚪ 可选 | 记录模型版本、参数、评估指标，毕设阶段可省略 |
| `forecast_task` 预测任务表 | ⚪ 可选 | 记录批量预测任务，batch_no 字段已能满足需求 |
| `feature_statistic` 特征统计表 | ❌ 不需要 | 数据量小，实时计算即可 |
| `prediction_history` 预测历史表 | ❌ 不需要 | forecast_result 本身就是历史记录 |

### 3.2 最终建议：一张表足够

**结论：预测模块只需要 `forecast_result` 一张核心表即可满足需求。**

原因：
1. **数据量可控**：每天约200个商品预测，一个月约6000条记录，单表完全能承载
2. **查询场景简单**：按日期查询、按商品查询、按分类查询，现有索引已覆盖
3. **批次管理已支持**：batch_no 字段可标识批量预测
4. **效果评估已支持**：actual_quantity 和 error_rate 字段支持模型验证

## 五、典型业务场景 SQL

### 5.1 执行批量预测并写入

```sql
-- 批量预测写入
INSERT INTO forecast_result (
    forecast_date, product_id,
    product_code, product_name, category_id, category_name,
    current_stock, safety_stock,
    weather, is_holiday, is_weekend,
    predicted_quantity, confidence,
    batch_no
)
SELECT
    '2026-03-28' AS forecast_date,
    p.id AS product_id,
    p.product_code,
    p.name AS product_name,
    p.category_id,
    pc.name AS category_name,
    p.stock AS current_stock,
    p.safety_stock,
    cf.weather,
    cf.is_holiday,
    cf.is_weekend,
    45 AS predicted_quantity,  -- 模型输出
    85 AS confidence,          -- 模型输出
    'BATCH-20260327-001' AS batch_no
FROM product p
LEFT JOIN product_category pc ON p.category_id = pc.id
LEFT JOIN calendar_factor cf ON cf.date = '2026-03-28'
WHERE p.status = 1;
```

### 5.2 查询某日预测结果（用于预测页面展示）

```sql
-- 单表查询，无需JOIN
SELECT
    id, forecast_date,
    product_code, product_name, category_name,
    current_stock, safety_stock,
    weather, is_holiday, is_weekend,
    predicted_quantity, confidence,
    actual_quantity, error_rate
FROM forecast_result
WHERE forecast_date = '2026-03-28'
ORDER BY category_name, product_name;
```

### 5.3 计算进货建议（预测结果 → 进货建议）

```sql
-- 筛选需要进货的商品
SELECT
    fr.id AS forecast_result_id,
    fr.product_id,
    fr.product_code,
    fr.product_name,
    fr.category_name,
    fr.predicted_quantity,
    fr.current_stock,
    fr.safety_stock,
    -- 计算建议进货量
    GREATEST(0, fr.predicted_quantity + fr.safety_stock - fr.current_stock) AS suggested_quantity
FROM forecast_result fr
WHERE fr.forecast_date = '2026-03-28'
  AND fr.predicted_quantity + fr.safety_stock > fr.current_stock;  -- 只返回需要进货的
```

### 5.4 更新实际销量和误差率（用于模型效果评估）

```sql
-- 预测日期过后，更新实际销量
UPDATE forecast_result fr
SET
    actual_quantity = (
        SELECT COALESCE(SUM(soi.quantity), 0)
        FROM sales_order_item soi
        WHERE soi.product_id = fr.product_id
        AND soi.sale_date = fr.forecast_date
    ),
    error_rate = CASE
        WHEN actual_quantity > 0
        THEN ROUND(ABS(predicted_quantity - actual_quantity) * 100.0 / actual_quantity, 2)
        ELSE NULL
    END
WHERE fr.forecast_date = '2026-03-28';
```

### 5.5 按批次查询预测记录

```sql
-- 查询某次批量预测的所有记录
SELECT
    batch_no,
    COUNT(*) AS total_products,
    SUM(predicted_quantity) AS total_predicted,
    AVG(confidence) AS avg_confidence,
    create_time
FROM forecast_result
WHERE batch_no = 'BATCH-20260327-001'
GROUP BY batch_no, create_time;
```

## 七、总结

### 7.1 最终结论

| 项目 | 结论 |
|------|------|
| **核心表** | `forecast_result` 一张表 |
| **新增字段** | `current_stock`、`safety_stock`、`batch_no` |
| **其他表** | 不需要，保持简洁 |

### 7.2 设计亮点

1. **单表设计**：通过冗余字段实现单表查询，无需 JOIN
2. **快照机制**：记录预测时的库存状态，数据可追溯
3. **批次标识**：batch_no 支持批量预测管理
4. **效果评估**：预留 actual_quantity 和 error_rate 支持模型验证
