### 3.2.1 用户角色分析

本系统面向中小型超市的日常经营管理，根据超市实际运营需求，系统设计了管理员和普通员工两类用户角色。不同角色具有不同的操作权限，以确保数据安全和职责分明。

**表3-1 用户角色权限划分**


| 角色   | 权限范围   | 主要功能                                            |
| ---- | ------ | ----------------------------------------------- |
| 管理员  | 系统全部功能 | 商品档案管理、销售数据导入、销售统计分析、库存监控、销量预测、进货建议、系统设置、操作日志查看 |
| 普通员工 | 基础业务操作 | 商品信息查询、销售流水查询、库存查询、个人信息管理                       |


**管理员职责**：

- 负责商品档案的维护，包括商品分类管理和商品信息的新增、编辑、删除
- 负责销售数据的导入和核对，确保数据准确性
- 查看销售统计报表，分析销售趋势，为经营决策提供依据
- 监控库存状态，处理库存预警，制定补货计划
- 管理系统用户，查看操作日志，保障系统安全

**普通员工职责**：

- 查询商品信息，了解商品库存和价格
- 查询销售流水，核对日常销售记录
- 查看库存状态，发现库存异常及时上报
- 管理个人账户信息，修改登录密码

> 注：当前版本为简化设计，所有登录用户具有管理员权限，暂不区分角色权限。后续版本可根据实际需求扩展RBAC权限管理模块。

### 3.2.2 功能模块划分

根据需求分析结果，本系统划分为六大功能模块，各模块功能描述如下：

**表3-2 系统功能模块划分**


| 模块名称    | 功能描述            | 主要子功能                       |
| ------- | --------------- | --------------------------- |
| 商品档案管理  | 管理商品分类和商品基础信息   | 商品分类管理、商品信息管理（新增/编辑/删除/查询）  |
| 销售数据中心  | 销售数据的导入、查询与统计分析 | 销售日结导入、销售流水查询、销售统计分析、日历因子管理 |
| 库存预警与监控 | 实时监控库存状态，提供预警功能 | 实时库存盘点、库存变动流水、库存预警设置        |
| 智能销量预测  | 基于历史数据的销量预测功能   | 单商品预测、批量预测、预测结果展示           |
| 智能补货建议  | 根据预测结果生成进货建议    | 进货建议计算、进货单生成                |
| 系统安全管理  | 用户认证与系统安全管理     | 用户登录认证、个人中心、密码修改、操作日志       |


**各模块详细功能说明**：

**1. 商品档案管理模块**

- 商品分类管理：支持两级分类结构，可新增、编辑、删除分类，支持分类排序和状态管理
- 商品信息管理：支持商品的新增、编辑、删除、查询，包含商品名称、条码、分类、进价、售价、库存、安全库存等字段，支持图片上传

**2. 销售数据中心模块**

- 销售日结导入：支持Excel格式的销售数据批量导入，自动解析订单和明细数据，支持数据校验和错误提示
- 销售流水查询：按日期范围、商品名称、分类等条件查询销售记录，支持分页展示和详情查看
- 销售统计分析：展示销售KPI指标（销售额、毛利、订单数、客单价），销售趋势折线图，品类销售占比饼图
- 日历因子管理：以日历视图展示每日信息，支持标记节假日、记录天气，为预测模型提供环境因子数据

**3. 库存预警与监控模块**

- 实时库存盘点：展示当前库存状态，支持按商品名称、分类、库存状态筛选，显示库存预警商品
- 库存变动流水：记录所有库存调整操作，包括入库、出库、调整等，支持按时间范围查询

**4. 智能销量预测模块**

- 单商品预测：选择单个商品，输入预测日期和环境因子，返回预测销量
- 批量预测：对多个商品同时进行预测，展示预测结果列表
- 预测结果展示：以表格和图表形式展示预测结果，包含预测销量、置信度、建议进货量

**5. 智能补货建议模块**

- 进货建议计算：根据预测销量、当前库存、安全库存计算建议进货量
- 进货单生成：支持一键生成进货单，可调整进货数量

**6. 系统安全管理模块**

- 用户登录认证：基于JWT Token的身份认证，支持登录、登出
- 个人中心：查看和修改个人信息（用户名、联系电话）
- 密码修改：验证原密码后修改新密码
- 操作日志：记录用户操作行为，包括操作时间、操作类型、操作描述、IP地址

### 3.3 非功能需求分析

#### 3.3.1 性能需求

系统需要满足中小型超市日常经营管理的性能要求，具体性能指标如下：

**表3-3 系统性能需求指标**


| 性能指标   | 需求描述          | 目标值                      |
| ------ | ------------- | ------------------------ |
| 页面响应时间 | 页面加载和操作响应速度   | 普通页面 ≤ 2秒，复杂查询 ≤ 5秒      |
| 接口响应时间 | 后端API接口处理时间   | 简单查询 ≤ 500ms，复杂统计 ≤ 3秒   |
| 数据导入性能 | Excel销售数据导入速度 | 1000条记录 ≤ 10秒            |
| 并发处理能力 | 系统同时处理请求的能力   | 支持10个用户同时在线操作            |
| 数据存储容量 | 系统数据存储规模      | 支持存储10000条商品记录，100万条销售记录 |


**性能保障措施**：

- 数据库查询优化：合理设计索引，对常用查询字段建立索引
- 分页查询：列表数据采用分页加载，避免一次性加载大量数据
- 异步处理：大数据量导入采用异步处理方式，避免阻塞主线程
- 缓存机制：对热点数据采用缓存，减少数据库访问压力

#### 3.3.2 安全性需求

系统需要保障数据安全和用户信息安全，具体安全需求如下：

**表3-4 系统安全性需求**


| 安全需求   | 需求描述          | 实现方案                                 |
| ------ | ------------- | ------------------------------------ |
| 身份认证   | 用户登录身份验证      | 采用JWT Token认证机制，Token有效期为24小时        |
| 访问控制   | 未登录用户无法访问系统资源 | 前端路由守卫拦截，后端拦截器校验Token                |
| 密码安全   | 用户密码安全存储      | 密码采用MD5加密存储，不明文存储                    |
| 数据传输安全 | 前后端数据传输安全     | 生产环境采用HTTPS协议加密传输                    |
| 操作审计   | 记录用户操作行为      | 采用AOP切面自动记录操作日志，包含操作人、操作类型、操作时间、IP地址 |
| 接口安全   | 防止接口被恶意调用     | 接口统一鉴权，敏感操作需验证Token有效性               |


**安全机制说明**：

- JWT认证流程：用户登录成功后，服务端生成JWT Token返回给前端；前端将Token存储在localStorage，每次请求通过Header携带Token；后端拦截器解析Token验证用户身份。
- 密码加密：用户注册和修改密码时，对密码进行MD5哈希处理后再存储到数据库，确保即使数据库泄露也无法直接获取用户密码。
- 操作日志：通过Spring AOP切面自动拦截Controller方法，记录用户操作行为，为安全审计提供依据。

#### 3.3.3 可维护性需求

系统需要具备良好的可维护性，便于后期维护和功能扩展，具体需求如下：

**表3-5 系统可维护性需求**


| 可维护性需求 | 需求描述            | 实现方案                                        |
| ------ | --------------- | ------------------------------------------- |
| 代码规范   | 代码风格统一，便于阅读和维护  | 遵循阿里巴巴Java开发规范，前端遵循Vue官方风格指南                |
| 模块解耦   | 模块间低耦合，便于独立修改   | 采用分层架构（Controller-Service-Mapper），模块按业务领域划分 |
| 代码复用   | 减少重复代码，提高开发效率   | 封装通用工具类和公共组件，如统一响应Result、分页工具等              |
| 文档完善   | 提供完整的开发文档       | 编写系统设计文档、接口文档、部署文档等                         |
| 日志记录   | 记录系统运行日志，便于问题排查 | 使用SLF4J+Logback记录日志，区分DEBUG、INFO、ERROR级别    |
| 配置外置   | 配置参数可配置，便于环境切换  | 采用application.yml配置文件，支持开发环境和生产环境配置分离       |


**架构设计保障**：

- 后端采用分层架构：Controller层负责接收请求和返回响应，Service层负责业务逻辑处理，Mapper层负责数据库访问，各层职责清晰，便于维护。
- 前端采用组件化开发：页面按功能模块划分组件，公共组件（如表格、表单、弹窗）统一封装，提高代码复用率。
- 统一异常处理：后端封装全局异常处理器，统一返回错误信息，避免代码中大量try-catch块。
