# 智能进货算法设计（简化版）

> 文档目的：用最简单的方式解决补货周期和分类进货问题
> 创建时间：2026年3月22日
> 原则：简单实用，符合中小型超市实际情况

---

## 一、核心问题

### 问题1：什么时候进货？进多少？

**现状**：只有"安全库存"，但这个概念模糊
- 安全库存是预警线还是订货点？
- 低于安全库存就进货？那进货到多少？

**实际需求**：
- 生鲜类：保质期短，库存够卖2-3天就行
- 饮料类：保质期长，库存够卖7-10天
- 日用品：保质期很长，库存够卖30天

### 问题2：如何避免频繁下单？

**现状**：每个商品单独判断是否进货
- 今天可乐缺了，下单
- 后天山泉缺了，下单
- 大后天雪碧缺了，下单

**实际需求**：
- 同类商品通常从同一供应商进货
- 应该一起检查、一起下单

## 二、简化设计方案

### 2.1 核心概念（只保留3个）

| 概念 | 含义 | 示例 |
|------|------|------|
| **补货周期** | 多久检查一次库存 | 生鲜1天，饮料7天，日用品30天 |
| **目标库存天数** | 库存够卖多少天 | 生鲜3天，饮料10天，日用品60天 |
| **日均销量** | 平均每天卖多少 | 近7天销量平均值 |

### 2.2 计算公式（小学生都能算）

```
目标库存 = 日均销量 × 目标库存天数

建议进货量 = 目标库存 - 当前库存
```

就这么简单，不需要服务水平、标准差、Z值这些复杂概念。

### 2.3 进货判断逻辑

```
检查时机：到了补货周期 OR 有商品库存紧张

判断规则：
  如果 当前库存 < 目标库存 × 50%  →  必须补货
  如果 当前库存 < 目标库存 × 80%  →  建议补货
  否则  →  不需要补货
```

**为什么用百分比？**
- 不同商品销量差异大，用绝对值不好统一
- 百分比更直观：库存只剩一半了，该补了

## 三、数据库设计

### 3.1 只改分类表，不改商品表

**原则**：商品表保持不变，策略放在分类表

```sql
-- 在 product_category 表增加2个字段
ALTER TABLE `product_category`
ADD COLUMN `review_cycle_days` INT DEFAULT 7 COMMENT '补货周期(天)',
ADD COLUMN `target_stock_days` INT DEFAULT 10 COMMENT '目标库存天数';
```

就这两个字段，解决周期问题。

### 3.2 分类配置示例

| 分类 | review_cycle_days | target_stock_days | 说明 |
|------|-------------------|-------------------|------|
| 生鲜果蔬 | 1 | 3 | 每天检查，库存够3天 |
| 饮料 | 7 | 10 | 每周检查，库存够10天 |
| 休闲零食 | 7 | 14 | 每周检查，库存够2周 |
| 粮油调味 | 14 | 30 | 半月检查，库存够1月 |
| 日用百货 | 30 | 60 | 每月检查，库存够2月 |

## 四、业务逻辑

### 4.1 生成进货建议流程

```
Step 1: 按分类检查
────────────────────
遍历每个分类：
  - 计算距离上次检查的天数
  - 如果 >= 补货周期，则检查该分类

Step 2: 分析每个商品
────────────────────
对于该分类下的每个商品：
  - 计算日均销量 = 近7天总销量 / 7
  - 计算目标库存 = 日均销量 × 目标库存天数
  - 计算库存比例 = 当前库存 / 目标库存

Step 3: 生成建议
────────────────────
如果 库存比例 < 50%：
  → 必须补货，建议进货量 = 目标库存 - 当前库存
如果 库存比例 < 80%：
  → 建议补货，建议进货量 = 目标库存 - 当前库存
否则：
  → 不需要补货

Step 4: 按分类汇总
────────────────────
如果该分类有"必须补货"的商品：
  → 将所有"必须补货"和"建议补货"的商品一起生成进货建议
  → 用户可以选择性采纳
```

### 4.2 代码示例（伪代码）

```java
/**
 * 生成某分类的进货建议
 */
public void generateRestockSuggestion(Long categoryId) {
    Category category = categoryMapper.selectById(categoryId);

    // 获取该分类下所有商品
    List<Product> products = productMapper.selectByCategoryId(categoryId);

    List<PurchaseSuggestion> suggestions = new ArrayList<>();
    boolean hasMustRestock = false;

    for (Product product : products) {
        // 计算日均销量（近7天）
        Double avgDailySales = salesMapper.getAvgDailySales(product.getId(), 7);

        // 计算目标库存
        int targetStock = (int) Math.ceil(avgDailySales * category.getTargetStockDays());

        // 计算库存比例
        double stockRatio = (double) product.getStock() / targetStock;

        // 判断是否需要补货
        if (stockRatio < 0.8) { // 低于80%
            PurchaseSuggestion suggestion = new PurchaseSuggestion();
            suggestion.setProductId(product.getId());
            suggestion.setCategoryId(categoryId);
            suggestion.setProductCode(product.getProductCode());
            suggestion.setProductName(product.getName());
            suggestion.setCategoryName(category.getName());
            suggestion.setPurchasePrice(product.getPurchasePrice());
            suggestion.setCurrentStock(product.getStock());
            suggestion.setAvgDailySales(avgDailySales);
            suggestion.setTargetStock(targetStock);

            int suggestedQty = targetStock - product.getStock();
            suggestion.setSuggestedQuantity(suggestedQty);
            suggestion.setFinalQuantity(suggestedQty);

            if (stockRatio < 0.5) {
                suggestion.setSuggestionType(1); // 必须补货
                hasMustRestock = true;
            } else {
                suggestion.setSuggestionType(2); // 建议补货
            }

            suggestions.add(suggestion);
        }
    }

    // 只有存在"必须补货"的商品时，才生成进货建议
    if (hasMustRestock && !suggestions.isEmpty()) {
        suggestionMapper.insertBatch(suggestions);
    }
}
```

## 七、总结

### 核心设计

1. **补货周期**：在分类表设置，不同分类不同周期
2. **目标库存**：日均销量 × 目标库存天数，简单直接
3. **分类进货**：有必须补货的商品时，整类一起建议

### 解决的问题

| 问题 | 解决方案 |
|------|----------|
| 不同商品周期不同 | 分类表设置 review_cycle_days |
| 进货标准不明确 | 目标库存 = 日均销量 × 目标天数 |
| 进货到什么水平 | 补到目标库存 |
| 频繁下单 | 按分类统一生成进货建议 |
