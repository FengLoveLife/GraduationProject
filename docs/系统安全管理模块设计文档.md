## 一、模块概述

### 1.1 设计背景

本系统定位为中小型超市管理系统，实际使用场景中，具有系统登录权限的用户数量很少（通常仅限几位店长），无需复杂的用户管理和权限分配机制。

### 1.2 设计决策

| 原计划 | 调整后 | 原因 |
|--------|--------|------|
| 用户管理 | ~~取消~~ | 用户数量少，无需复杂的增删改查 |
| 个人中心 | 保留 | 用户查看/修改个人信息、密码 |
| 操作日志 | 新增 | 记录关键操作，答辩加分项 |

### 1.3 模块定位

```
系统安全管理
├── 个人中心    ← 查看/修改个人信息、修改密码
└── 操作日志    ← 记录关键操作（可追溯）
```

### 2.4 后端 DTO 设计

```java
/**
 * 修改个人信息 DTO
 */
@Data
public class UpdateProfileDTO {
    @NotBlank(message = "真实姓名不能为空")
    @Size(max = 20, message = "真实姓名不能超过20个字符")
    private String realName;
}

/**
 * 修改密码 DTO
 */
@Data
public class UpdatePasswordDTO {
    @NotBlank(message = "原密码不能为空")
    private String oldPassword;

    @NotBlank(message = "新密码不能为空")
    @Size(min = 6, max = 20, message = "密码长度需在6-20位之间")
    private String newPassword;

    @NotBlank(message = "确认密码不能为空")
    private String confirmPassword;
}
```

## 三、操作日志

### 3.1 功能说明

操作日志用于记录系统中的关键操作，便于追溯和审计。对于毕业答辩来说，这是一个展示系统安全设计意识的加分项。

### 3.2 记录的操作类型

| 操作类型 | 类型编码 | 记录时机 |
|----------|----------|----------|
| 登录 | LOGIN | 登录成功/失败 |
| 密码 | PASSWORD | 修改密码成功 |
| 商品 | PRODUCT | 新增/修改/删除商品 |
| 销售 | SALES | 导入销售数据 |
| 库存 | INVENTORY | 库存盘点调整 |

### 3.4 数据库设计

```sql
CREATE TABLE sys_operation_log (
    id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键',
    user_id BIGINT NOT NULL COMMENT '操作用户ID',
    user_name VARCHAR(50) NOT NULL COMMENT '操作用户名',
    operation_type VARCHAR(20) NOT NULL COMMENT '操作类型：LOGIN/PASSWORD/PRODUCT/SALES/INVENTORY',
    operation_desc VARCHAR(200) COMMENT '操作描述',
    ip_address VARCHAR(50) COMMENT 'IP地址',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '操作时间',
    INDEX idx_user_id (user_id),
    INDEX idx_type (operation_type),
    INDEX idx_create_time (create_time)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='操作日志表';
```

### 3.6 后端实体设计

```java
@Data
@TableName("sys_operation_log")
public class SysOperationLog implements Serializable {

    @TableId(value = "id", type = IdType.AUTO)
    private Long id;

    /** 操作用户ID */
    private Long userId;

    /** 操作用户名 */
    private String userName;

    /** 操作类型 */
    private String operationType;

    /** 操作描述 */
    private String operationDesc;

    /** IP地址 */

    /** 操作时间 */

```

### 3.7 日志记录实现方式

采用 AOP 切面 + 注解的方式，在关键操作上添加注解自动记录日志：

```java
/**
 * 操作日志注解
 */
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface OperationLog {
    String type();      // 操作类型
    String desc();      // 操作描述
}

/**
 * 使用示例
 */
@PostMapping("/product")
@OperationLog(type = "PRODUCT", desc = "新增商品")
public Result<Product> addProduct(@RequestBody ProductDTO dto) {
    // ...
}
```

## 四、数据库扩展

### 4.1 sys_user 表扩展字段

当前表结构缺少部分字段，建议扩展：

```sql
ALTER TABLE sys_user
ADD COLUMN last_login_time DATETIME COMMENT '最后登录时间',
ADD COLUMN phone VARCHAR(20) COMMENT '联系电话',
ADD COLUMN update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间';
```

### 4.2 完整表结构

```sql
CREATE TABLE sys_user (
    id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键',
    username VARCHAR(50) NOT NULL UNIQUE COMMENT '用户名',
    password VARCHAR(100) NOT NULL COMMENT '密码',
    real_name VARCHAR(50) COMMENT '真实姓名',
    status VARCHAR(20) DEFAULT 'ACTIVE' COMMENT '状态：ACTIVE/INACTIVE',
    phone VARCHAR(20) COMMENT '联系电话',
    last_login_time DATETIME COMMENT '最后登录时间',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='系统用户表';
```

## 七、注意事项

1. **密码安全**：当前系统使用明文/MD5存储密码，仅适合演示。生产环境应使用 BCrypt。

2. **日志清理**：操作日志表会持续增长，建议定期清理（如保留最近3个月数据）或在表中增加定时任务清理。

3. **IP地址获取**：如系统部署在反向代理后，需从 `X-Forwarded-For` 头获取真实IP。
