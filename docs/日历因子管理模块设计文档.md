## 一、模块概述

### 1.1 什么是日历因子？

**日历因子 = 影响销量的外部环境因素**

在零售行业，商品销量不仅仅由商品本身决定，还受到许多外部环境因素的影响。这些因素包括：

- **时间规律**：周末销量通常比工作日高
- **节假日效应**：春节、国庆等节假日前夕销量暴增
- **天气影响**：下雨天客流减少，炎热天气饮料销量上升
- **特殊日期**：情人节鲜花热销，开学季文具畅销

### 1.2 为什么对AI预测重要？

回顾我们的销售数据模拟公式：

```
最终销量 = 基础销量 × 时间因子 × 天气因子 × 节假日因子 × 随机扰动
```

**如果没有日历因子数据，AI模型将无法学习到这些规律：**

| 缺失因子 | 模型无法学到的规律 |
|----------|-------------------|
| 星期因子 | 不知道周末销量比工作日高 |
| 天气因子 | 不知道下雨天饮料卖不动 |
| 节假日因子 | 不知道春节前销量会暴增 |

**日历因子表的作用：告诉模型"今天是什么特殊日子"的标签数据。**

## 二、数据库设计

### 2.1 现有表结构

```sql
CREATE TABLE `calendar_factor` (
    `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '主键ID',
    `date` DATE NOT NULL COMMENT '日期(唯一)',
    `day_of_week` TINYINT NOT NULL COMMENT '星期几 1-7(1=周一)',
    `is_weekend` TINYINT NOT NULL DEFAULT 0 COMMENT '是否周末: 0否 1是',
    `is_holiday` TINYINT NOT NULL DEFAULT 0 COMMENT '是否节假日: 0否 1是',
    `holiday_name` VARCHAR(50) DEFAULT NULL COMMENT '节假日名称: 春节等',
    `weather` VARCHAR(20) DEFAULT NULL COMMENT '天气: 晴/多云/雨/雪',
    `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    `update_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_date` (`date`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='日历影响因子表';
```

### 2.2 字段详细说明

| 字段名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| `id` | BIGINT | 是 | 自增 | 主键 |
| `date` | DATE | 是 | - | 日期，唯一键，不可重复 |
| `day_of_week` | TINYINT | 是 | - | 星期几，1=周一，7=周日，系统自动计算 |
| `is_weekend` | TINYINT | 是 | 0 | 是否周末，系统自动计算 |
| `is_holiday` | TINYINT | 是 | 0 | 是否节假日，手动标记 |
| `holiday_name` | VARCHAR(50) | 否 | NULL | 节假日名称，如"春节"、"国庆" |
| `weather` | VARCHAR(20) | 否 | NULL | 天气状态，枚举值 |
| `create_time` | DATETIME | 是 | 当前时间 | 创建时间 |
| `update_time` | DATETIME | 是 | 当前时间 | 更新时间 |

### 2.3 天气枚举值规范

| 天气值 | 中文 | 图标 | 对销量的影响 |
|--------|------|------|--------------|
| `晴` | 晴天 | ☀️ | 正常 |
| `多云` | 多云 | ☁️ | 正常 |
| `小雨` | 小雨 | 🌧️ | 客流减少，销量下降15% |
| `大雨` | 大雨 | ⛈️ | 客流大幅减少，销量下降40% |
| `雪` | 下雪 | ❄️ | 客流减少 |
| `炎热` | 炎热 | 🌡️ | 饮料销量上升 |

## 三、核心功能设计

### 3.1 功能清单

| 序号 | 功能名称 | 优先级 | 说明 |
|------|----------|--------|------|
| 1 | 日历视图展示 | **高** | 以月历形式展示所有日期的因子状态 |
| 2 | 批量初始化 | **高** | 一键生成未来N天的日历数据 |
| 3 | 节假日标记 | **高** | 手动标记/取消节假日 |
| 4 | 天气录入 | 中 | 为日期设置天气状态 |
| 5 | 快速编辑 | 中 | 点击日期弹出编辑框 |
| 6 | 月份切换 | 中 | 按月份查看日历数据 |
| 7 | 批量设置天气 | 低 | 为连续多天设置相同天气 |
| 8 | 节假日导入 | 低 | 从法定节假日文件导入 |

### 3.2 功能详细描述

#### 3.2.1 日历视图展示

**功能描述**：以日历形式展示一个月内所有日期的日历因子状态。

**展示内容**：
- 日期数字（如：1, 2, 3...）
- 周末标记（自动识别周六周日）
- 节假日标记（手动设置）
- 天气图标（晴、雨、雪等）

**视觉区分**：
- 周末：浅色背景或标签
- 节假日：红色标签 + 节假日名称
- 天气：对应图标显示

#### 3.2.2 批量初始化

**功能描述**：根据日期范围，批量生成日历因子记录。

**生成规则**：
- 自动计算 `day_of_week`（星期几）
- 自动设置 `is_weekend`（是否周末）
- `is_holiday` 默认为 0
- `weather` 默认为 NULL

**使用场景**：系统首次使用时，需要生成基础日历数据。

#### 3.2.3 节假日标记

**功能描述**：手动将某个日期标记为节假日。

**操作方式**：
- 点击日期，弹出编辑框
- 勾选"是否节假日"
- 输入节假日名称（如：春节、国庆）
- 保存更新

**业务意义**：节假日的销量通常比普通日高，模型需要这个特征。

#### 3.2.4 天气录入

**功能描述**：为日期设置天气状态。

**操作方式**：
- 点击日期，选择天气（晴/多云/雨/雪等）
- 保存更新

**业务意义**：
- **历史天气**：用于模型训练（已过去的日期）
- **未来天气**：用于预测输入（用户手动设置预测日期的天气）

## 九、注意事项

### 9.1 数据一致性

- `day_of_week` 和 `is_weekend` 必须与 `date` 一致
- 系统自动计算，不允许手动修改
- 初始化时需校验日期范围合法性

### 9.2 唯一性约束

- `date` 字段有唯一索引
- 初始化时跳过已存在的日期
- 更新时不允许修改日期

### 9.3 天气数据来源

- **历史日期**：可从天气API获取（可选）
- **未来日期**：由用户手动设置，用于预测

### 9.4 性能考虑

- 一个月最多31条记录，查询性能无问题
- 批量初始化时注意控制日期范围（建议不超过1年）
