## 一、核心业务指标（KPI卡片）

这是统计分析的"门面"，展示关键数字：

| 指标 | 计算方式 | 业务意义 |
|------|----------|----------|
| 总销售额 | SUM(订单金额) | 营收规模 |
| 总毛利润 | SUM(明细毛利) | 实际盈利 |
| 毛利率 | 毛利润/销售额 | 盈利能力 |
| 订单数 | COUNT(订单) | 交易活跃度 |
| 客单价 | 销售额/订单数 | 消费水平 |
| 商品销量 | SUM(销售数量) | 流通规模 |

## 五、后端接口设计

### 5.1 接口列表

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/sales-order/statistics/overview` | GET | 概览数据(KPI卡片) |
| `/api/sales-order/statistics/trend` | GET | 销售趋势数据 |
| `/api/sales-order/statistics/top-products` | GET | 商品销量TOP10 |
| `/api/sales-order/statistics/category` | GET | 分类销售统计 |
| `/api/sales-order/statistics/payment` | GET | 支付方式统计 |

### 5.2 接口参数设计

**概览数据接口**：
```
GET /api/sales-order/statistics/overview
参数：
  - startDate: 开始日期 (必填)
  - endDate: 结束日期 (必填)
  - categoryId: 分类ID (可选)

返回：
{
  "totalAmount": 12580.00,      // 总销售额
  "totalProfit": 3420.00,       // 总毛利
  "profitRate": 27.19,          // 毛利率(%)
  "orderCount": 156,            // 订单数
  "avgOrderAmount": 80.64,      // 客单价
  "totalQuantity": 892          // 总销量
}
```

**趋势数据接口**：
```
GET /api/sales-order/statistics/trend
参数：
  - startDate: 开始日期 (必填)
  - endDate: 结束日期 (必填)
  - type: 粒度类型 (day/week/month，默认day)
  - categoryId: 分类ID (可选)

返回：
{
  "dates": ["03-01", "03-02", "03-03", ...],
  "amounts": [1200, 1350, 980, ...],
  "profits": [320, 380, 260, ...],
  "orderCounts": [15, 18, 12, ...]
}
```

```
GET /api/sales-order/statistics/top-products

返回：

    { "productId": 1, "productName": "可口可乐", "quantity": 320, "amount": 1920.00 },
```

## 八、SQL参考

### 8.1 概览统计

```sql
-- 总销售额、总毛利、订单数、总销量
SELECT
    COALESCE(SUM(so.total_amount), 0) as total_amount,
    COALESCE(SUM(so.total_amount) - SUM(soi.purchase_price * soi.quantity), 0) as total_profit,
    COUNT(DISTINCT so.id) as order_count,
    COALESCE(SUM(soi.quantity), 0) as total_quantity
FROM sales_order so
LEFT JOIN sales_order_item soi ON so.id = soi.order_id
WHERE so.sale_date BETWEEN #{startDate} AND #{endDate}
```

### 8.2 趋势统计（按日）

```sql
SELECT
    so.sale_date as date,
    SUM(so.total_amount) as amount,
    SUM(soi.subtotal_profit) as profit,
    COUNT(DISTINCT so.id) as order_count
FROM sales_order so
LEFT JOIN sales_order_item soi ON so.id = soi.order_id
WHERE so.sale_date BETWEEN #{startDate} AND #{endDate}
GROUP BY so.sale_date
ORDER BY so.sale_date
```

### 8.3 TOP商品

```sql
SELECT
    soi.product_id,
    soi.product_name,
    SUM(soi.quantity) as quantity,
    SUM(soi.subtotal_amount) as amount
FROM sales_order_item soi
JOIN sales_order so ON soi.order_id = so.id
WHERE so.sale_date BETWEEN #{startDate} AND #{endDate}
GROUP BY soi.product_id, soi.product_name
ORDER BY quantity DESC
LIMIT #{limit}
```

### 8.4 分类统计

```sql
SELECT
    soi.category_id,
    soi.category_name,
    SUM(soi.subtotal_amount) as amount
FROM sales_order_item soi
JOIN sales_order so ON soi.order_id = so.id
WHERE so.sale_date BETWEEN #{startDate} AND #{endDate}
GROUP BY soi.category_id, soi.category_name
ORDER BY amount DESC
```

### 8.5 支付方式统计

```sql
SELECT
    so.payment_type,
    SUM(so.total_amount) as amount,
    COUNT(*) as order_count
FROM sales_order so
WHERE so.sale_date BETWEEN #{startDate} AND #{endDate}
GROUP BY so.payment_type
ORDER BY amount DESC
```

## 九、注意事项

1. **数据为空处理**：使用 COALESCE 函数避免 NULL 值
2. **日期范围校验**：后端需校验日期格式和范围合理性
3. **性能优化**：大数据量时考虑添加索引或缓存
4. **权限控制**：统计接口需要登录验证
