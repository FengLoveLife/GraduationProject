### 🐞 唯一的小瑕疵：登录日志会变成“未知用户”

**问题出在哪里？**
你在 `SysUserController` 的 `login` 方法上优雅地加上了 `@OperationLog(type = "LOGIN", desc = "登录成功")`。
但是，当程序运行到 `OperationLogAspect` 切面去记录日志时，它是通过以下代码获取当前用户的：
```java
Object claimsObj = request.getAttribute("jwtClaims");
```
**问题来了**：用户在调用 `/login` 接口时，是还没有 Token 的（拦截器会放行登录接口），所以此时 request 里根本没有 `jwtClaims`！
这就会导致 AOP 切面抓不到用户信息，最终存进数据库的登录日志，操作人全部都会变成兜底的 **"未知用户"**。

### 🛠️ 极其简单的修复方案

对于登录这种极其特殊的接口（它是在生成身份，而不是携带身份），最优雅的解法是**不在登录接口上使用 AOP，而是在登录成功后手动记录一行日志**。

请对 `src/main/java/com/saul/user/controller/SysUserController.java` 做如下微调：

**1. 删掉 `login` 方法上的 `@OperationLog` 注解：**
```java
    /**
     * 登录接口
     */
    @PostMapping("/login")
    // 删掉这行：@OperationLog(type = "LOGIN", desc = "登录成功")
    public Result<Map<String, Object>> login(@RequestBody LoginDTO dto, HttpServletRequest request) {
```

**2. 在 `login` 方法内部，Token 生成之后，手动插入日志：**
在 `sysUserService.updateLastLoginTime(user.getId());` 这一行的下面，加上这两句：
```java
            // 获取客户端IP
            String ipAddress = request.getRemoteAddr(); // 简单获取即可，如果需要精确可复用获取IP的工具方法
            
            // 手动记录登录日志
            logService.recordLog(user.getId(), user.getRealName(), "LOGIN", "登录系统成功", ipAddress);
```
