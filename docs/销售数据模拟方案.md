# 销售数据模拟方案

## 一、核心思路

### 1.1 设计原则

模拟数据必须具备**可预测性**，即数据中包含可被算法识别的规律。我们的设计思路：

```
实际销量 = 基础销量 × 时间因子 × 天气因子 × 节假日因子 × 随机波动
```

- **基础销量**：每个商品有一个"正常情况下的日均销量"
- **时间因子**：周末、月底、季节等时间因素
- **天气因子**：温度、降水对销量的影响
- **节假日因子**：春节、国庆、周末等
- **随机波动**：±10%~20% 的随机变化，模拟真实的不确定性

### 1.2 数据规模建议

| 时间范围 | 数据量 | 说明 |
|---------|-------|------|
| 历史 3 个月 | 约 10-15 万条订单明细 | 用于模型训练和系统展示 |
| 未来 7 天预测 | - | 基于历史数据预测 |

## 二、影响因子设计

### 2.1 时间因子

#### 2.1.1 星期因子（week_factor）
| 星期 | 因子值 | 说明 |
|-----|-------|------|
| 周一至周四 | 0.9 | 工作日，销量一般 |
| 周五 | 1.1 | 周末前夕，采购增加 |
| 周六 | 1.3 | 周末高峰 |
| 周日 | 1.2 | 周末，略低于周六 |

#### 2.1.2 月份因子（month_factor）
| 月份 | 因子值 | 说明 |
|-----|-------|------|
| 1月 | 1.2 | 年货采购高峰 |
| 2月 | 0.8 | 春节假期，部分店铺歇业 |
| 3月 | 1.0 | 正常 |
| 4月 | 1.0 | 正常 |
| 5月 | 1.1 | 五一假期 |
| 6月 | 1.1 | 端午节、儿童节 |
| 7-8月 | 1.2 | 暑期，饮料、冷饮销量高 |
| 9月 | 1.0 | 正常 |
| 10月 | 1.2 | 国庆长假 |
| 11月 | 1.1 | 双十一促销 |
| 12月 | 1.3 | 年底采购、圣诞元旦 |

#### 2.1.3 月底因子（month_end_factor）
- 每月 25-31 日：因子 = 1.15（发工资后采购增加）

### 2.2 天气因子

#### 2.2.1 温度因子（temp_factor）

按商品类别区分影响：

| 商品类别 | 温度区间 | 因子值 | 说明 |
|---------|---------|-------|------|
| 饮料/冷饮 | >30℃ | 1.5 | 高温，销量大增 |
| 饮料/冷饮 | 20-30℃ | 1.2 | 温暖，销量较好 |
| 饮料/冷饮 | 10-20℃ | 0.8 | 凉爽，销量下降 |
| 饮料/冷饮 | <10℃ | 0.5 | 寒冷，销量锐减 |
| 方便速食 | 任意 | 1.0 | 不受温度影响 |
| 休闲零食 | 任意 | 1.0 | 不受温度影响 |
| 日化用品 | 任意 | 1.0 | 不受温度影响 |

#### 2.2.2 降水因子（rain_factor）

| 天气类型 | 因子值 | 说明 |
|---------|-------|------|
| 晴天/多云 | 1.0 | 正常 |
| 小雨 | 0.85 | 出行减少 |
| 中雨 | 0.7 | 明显减少 |
| 大雨/暴雨 | 0.5 | 严重减少 |
| 雪天 | 0.6 | 雪天出行困难 |

### 2.3 节假日因子（holiday_factor）

| 节假日 | 因子值 | 说明 |
|-------|-------|------|
| 春节前7天 | 1.5 | 年货采购高峰 |
| 春节期间 | 0.6 | 部分歇业 |
| 清明节 | 0.9 | 略有下降 |
| 五一假期 | 1.2 | 度假采购 |
| 端午节 | 1.1 | 粽子等传统商品 |
| 中秋节 | 1.1 | 月饼等传统商品 |
| 国庆节 | 1.3 | 长假消费 |
| 双十一 | 1.3 | 促销活动 |
| 双十二 | 1.1 | 促销活动 |
| 圣诞节 | 1.05 | 年轻人消费 |
| 元旦 | 1.1 | 新年消费 |
| 普通日期 | 1.0 | 正常 |

### 2.4 商品特性因子

每个商品有基础销量和季节特性：

| 商品类别 | 基础日销量范围 | 季节特性 |
|---------|--------------|---------|
| 饮料 | 10-50 件 | 夏季高峰，冬季低谷 |
| 冷饮/冰淇淋 | 5-30 件 | 夏季热销，冬季几乎不卖 |

### 4.2 方案二：模拟天气数据（简单可行）

由于你只需要模拟数据，可以直接根据日期生成合理的模拟天气：

```java
/**
 * 根据日期生成模拟天气数据
 * 以长沙地区为例（可根据实际城市调整）
 */
public WeatherData generateWeather(LocalDate date) {
    int month = date.getMonthValue();

    // 1. 生成温度（根据月份）
    int temp = generateTemperature(month);

    // 2. 生成天气类型（根据温度和月份）
    String weatherType = generateWeatherType(month, temp);

    // 3. 计算影响因子
    BigDecimal tempFactor = calculateTempFactor(temp);
    BigDecimal rainFactor = calculateRainFactor(weatherType);

    return new WeatherData(date, temp, weatherType, tempFactor, rainFactor);
}

private int generateTemperature(int month) {
    // 长沙地区月平均温度范围
    int[] avgTemp = {5, 8, 12, 18, 24, 28, 31, 30, 26, 20, 14, 8};
    int[] range = {8, 10, 12, 12, 10, 8, 6, 6, 10, 12, 12, 10};

    int base = avgTemp[month - 1];
    int offset = (int) (Math.random() * range[month - 1]) - range[month - 1] / 2;
    return base + offset;
}

private String generateWeatherType(int month, int temp) {
    // 根据月份决定降水概率
    double[] rainProb = {0.3, 0.35, 0.4, 0.45, 0.5, 0.55, 0.45, 0.4, 0.35, 0.35, 0.35, 0.3};

    double rand = Math.random();
    if (rand < rainProb[month - 1] * 0.3) {
        return "大雨";
    } else if (rand < rainProb[month - 1] * 0.7) {
        return "中雨";
    } else if (rand < rainProb[month - 1]) {
        return "小雨";
    } else if (temp > 30) {
        return "晴";
    } else {
        return Math.random() > 0.5 ? "晴" : "多云";
    }
}
```

## 五、模拟算法实现

### 5.1 核心算法流程

```
步骤1：生成日历因子数据
   ↓
步骤2：生成天气数据并计算天气因子
   ↓
步骤3：为每个商品设置基础销量和季节类型
   ↓
步骤4：遍历每一天，计算当日各商品预期销量
   ↓
步骤5：根据预期销量生成订单和订单明细
   ↓
步骤6：插入数据库
```

### 5.2 日销量计算公式

```java
/**
 * 计算某商品在某天的预期销量
 */
public int calculateExpectedSales(
    Product product,           // 商品信息
    LocalDate date,            // 日期
    CalendarFactor factor,     // 日历因子
    WeatherData weather        // 天气数据
) {
    // 1. 获取基础销量
    int baseSales = product.getBaseDailySales();

    // 2. 获取各类因子
    double weekFactor = factor.getWeekFactor();        // 星期因子
    double monthFactor = factor.getMonthFactor();      // 月份因子
    double holidayFactor = factor.getHolidayFactor();  // 节假日因子

    // 3. 获取天气因子（根据商品是否温度敏感）
    double tempFactor = 1.0;
    double rainFactor = weather.getRainFactor();

    if (product.isTempSensitive()) {
        tempFactor = weather.getTempFactor();
    }

    // 4. 计算季节因子
    double seasonFactor = calculateSeasonFactor(product.getSeasonType(), date.getMonthValue());

    // 5. 综合计算
    double expectedSales = baseSales
        * weekFactor
        * monthFactor
        * holidayFactor
        * tempFactor
        * rainFactor
        * seasonFactor;

    // 6. 添加随机波动（±15%）
    double randomFactor = 0.85 + Math.random() * 0.30;
    expectedSales *= randomFactor;

    // 7. 返回整数销量
    return Math.max(0, (int) Math.round(expectedSales));
}

/**
 * 计算季节因子
 */
private double calculateSeasonFactor(String seasonType, int month) {
    if ("summer_peak".equals(seasonType)) {
        // 夏季高峰型（饮料、冷饮）
        if (month >= 6 && month <= 8) {
            return 1.4;
        } else if (month >= 4 && month <= 5) {
            return 1.1;
        } else if (month >= 9 && month <= 10) {
            return 0.9;
        } else {
            return 0.6;
        }
    } else if ("winter_peak".equals(seasonType)) {
        // 冬季高峰型（火锅食材、保暖用品）
        if (month >= 11 || month <= 2) {
            return 1.3;
        } else if (month >= 3 && month <= 4) {
            return 0.9;
        } else {
            return 0.7;
        }
    } else {
        // 普通型
        return 1.0;
    }
}
```

### 6.2 代码配置方式

```java
public class HolidayConfig {

    private static final Map<String, Double> HOLIDAY_FACTORS = new HashMap<>();
    private static final Map<LocalDate, HolidayInfo> HOLIDAY_MAP = new HashMap<>();

    static {
        // 配置节假日因子
        HOLIDAY_FACTORS.put("spring_festival", 0.6);
        HOLIDAY_FACTORS.put("spring_festival_before", 1.5);
        HOLIDAY_FACTORS.put("national_day", 1.3);
        HOLIDAY_FACTORS.put("new_year", 1.1);
        // ... 其他节假日

        // 配置具体日期
        // 2024年春节
        putHolidayRange("2024-02-03", "2024-02-09", "spring_festival_before", "春节前");
        putHolidayRange("2024-02-10", "2024-02-17", "spring_festival", "春节");
        // ... 其他日期
    }

    private static void putHolidayRange(String start, String end,
                                        String type, String name) {
        LocalDate startDate = LocalDate.parse(start);
        LocalDate endDate = LocalDate.parse(end);
        double factor = HOLIDAY_FACTORS.get(type);

        for (LocalDate date = startDate; !date.isAfter(endDate); date = date.plusDays(1)) {
            HOLIDAY_MAP.put(date, new HolidayInfo(factor, name, true));
        }
    }

    public static HolidayInfo getHolidayInfo(LocalDate date) {
        return HOLIDAY_MAP.getOrDefault(date,
            new HolidayInfo(1.0, "普通日期", false));
    }
}
```

## 八、验证数据质量

生成数据后，需要验证数据是否具备可预测性：

```sql
-- 查看每日销售趋势
SELECT sale_date, SUM(total_amount) as daily_sales
FROM sales_order
GROUP BY sale_date
ORDER BY sale_date;

-- 查看周末vs工作日销量对比
SELECT
    DAYOFWEEK(sale_date) as weekday,
    AVG(total_amount) as avg_sales
FROM sales_order
GROUP BY DAYOFWEEK(sale_date);

-- 查看各月销量对比
SELECT
    MONTH(sale_date) as month,
    SUM(total_amount) as total_sales
FROM sales_order
GROUP BY MONTH(sale_date);
```

如果数据具备以下特征，说明模拟成功：
- ✅ 周末销量明显高于工作日
- ✅ 节假日前后有明显的销量波动
- ✅ 季节性商品（饮料）夏季销量高
- ✅ 整体有合理的波动，但不过于剧烈

## 九、后续优化建议

1. **增加促销活动因子**：可模拟促销日活动，销量提升 1.5-2 倍
2. **增加时间段分布**：早中晚不同时段订单分布
3. **增加客户类型因子**：VIP客户、散客等
4. **增加异常数据**：偶尔的库存不足、系统故障等，更真实

---

**总结**：模拟数据的关键是**规律性 + 随机性**的平衡。规律太明显预测效果太好（不真实），随机性太强无法预测。通过多因子叠加 + 适度随机，可以生成既真实又有规律的数据。
