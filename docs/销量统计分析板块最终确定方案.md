
### 🏆 终极版【销售统计分析】架构方案

我们将整个大屏分为 **“控制台 -> 核心指标 -> 趋势画像 -> 结构画像”** 四个层次：

#### 1. 全局控制台 (Global Filter)

* **组件**：带快捷选项（今日、昨日、近7天、近30天、本月）的日期范围选择器。
* **联动**：改变时间，下方所有图表无刷新重新渲染。

#### 2. 四大核心 KPI 卡片 (高信息密度设计)

融合了 Claude 的建议，我们将单调的卡片升级为“主副指标”结构：

* 💰 **卡片1：营业收入**。主指标：`总销售额`；副指标：`总订单数`。
* 📈 **卡片2：盈利情况**。主指标：`总毛利润`；副指标：`综合毛利率` (毛利润 ÷ 销售额)。 (大老板最看重这个，通常超市应该在 20%-30% 左右)
* 🧑‍🤝‍🧑 **卡片3：客单分析**。主指标：`客单价` (销售额 ÷ 订单数)；副指标：比较上周期的`环比增幅`。
* 🛒 **卡片4：动销情况**。主指标：`商品总销量 (件)`。

#### 3. 核心趋势画像 (Trend Analysis)

* **图表**：双 Y 轴宽幅折线图 (Echarts `line`)。
* **数据**：左侧 Y 轴是 `销售额` (蓝线)，右侧 Y 轴是 `毛利` (绿线)，X 轴是 `日期`。
* **✨ 架构师专属加分项**：在后端查询时，顺便把 `calendar_factor` 的数据查出来。如果在图表上碰到“周末”或“节假日”，在折线的波峰上用一个小小的红点（`markPoint`）标出来。答辩时你可以骄傲地指着这里说：“看，这就是节假日因子的直观体现！”

#### 4. 三维结构画像 (占比与排行)

采用一行三列（`el-col :span="8"`）的布局，完美融入 Claude 的“支付方式”：

* 🏆 **左侧：商品销量 TOP 10** (横向柱状图)。一眼看出店里的“引流爆款”。
* 🍰 **中间：品类销售占比** (南丁格尔玫瑰图 / 环形图)。展示“生鲜”、“零食”哪个更赚钱。
* 💳 **右侧：支付方式偏好** (饼图)。直观展示现金、微信、支付宝的比例，方便店长了解收款结构。

### 💻 后端实现路径（极致性能挑战）

正如 Claude 给出的 SQL 示例，这几个图表其实**不需要在 Java 里写任何循环计算**。 我们要做的，是极其优雅地使用 MyBatis-Plus 的自定义 XML，把算力交给 MySQL。

我们需要在后端建立以下几个接口：

1. `GET /api/sales-analysis/kpi` -> 返回 KPI 四大指标。
2. `GET /api/sales-analysis/trend` -> 返回每天的营业额和毛利数组。
3. `GET /api/sales-analysis/top-products` -> 返回 TOP 10。
4. `GET /api/sales-analysis/category-pie` -> 返回分类饼图数据。
5. `GET /api/sales-analysis/payment-pie` -> 返回支付方式饼图数据。
